# ClutchReframe 域名 + 多站点架构方案

更新日期：2026-06-17

## 1. 结论

ClutchReframe 继续采用一个主域名 + 多子域名的结构，但需要把 **产品官网 / Riot 审核站点** 和 **Overwolf Store Listing Support URL** 隔离。

当前推荐结构：

| 子域名 | 站点用途 | 仓库 | 是否需要 `riot.txt` |
| --- | --- | --- | --- |
| `clips.clutchreframe.com` | ClutchReframe Clips 官网 / Riot 或 LoL 相关审核材料 | `ClutchReframe/Clips-website` | 是，若用于 Riot 审核 |
| `live.clutchreframe.com` | ClutchReframe Live 官网 / Riot 或 LoL 相关审核材料 | `ClutchReframe/Live-website` | 是，若用于 Riot 审核 |
| `support.clutchreframe.com` | ClutchReframe Clips Overwolf Store Listing Support link | `ClutchReframe/Support-website` | 否 |

关键规则：

- `clips.clutchreframe.com` 已用于 Riot / LoL 相关审核，不要混入当前 Dota 2-only Overwolf support 内容。
- `support.clutchreframe.com` 只作为 Overwolf Store Listing 的 Support URL，不做产品官网。
- Support 站不放 `riot.txt`，不提交 Riot 审核，不写 LoL / Riot support 承诺。
- 当前 Overwolf OPK 口径是 Dota 2-only，因此 support 页面只写 Dota 2 support。

## 2. 背景

品牌统一为 `ClutchReframe`，产品线包括：

- `ClutchReframe Clips`：赛后高光捕捉与竖屏剪辑。
- `ClutchReframe Live`：直播运镜方向，后续独立推进。

此前域名方案主要服务 Clips / Live 两个产品线的 Riot 审核，因此重点是每个站点独立放置 `riot.txt`。现在 Overwolf Dota 2-only OPK 还需要一个真实可公开访问的 Support link，用于 Store Listing 中的用户技术支持入口。

因为 `clips.clutchreframe.com` 已经用于 Riot / LoL 审核，Support link 应使用独立子域名：

```text
https://support.clutchreframe.com/
```

## 3. 站点职责边界

### 3.1 `clips.clutchreframe.com`

用途：

- ClutchReframe Clips 产品官网。
- Riot / LoL 相关审核材料。
- 可包含 `riot.txt`、privacy、terms、OG metadata、产品描述。

限制：

- 审核期间不要随意改变 URL 结构。
- 不作为 Dota 2-only Overwolf Store Listing 的 support 入口。
- 不把 support 站的 Dota 2-only 排障内容混入该站。

### 3.2 `live.clutchreframe.com`

用途：

- ClutchReframe Live 产品官网。
- Live 产品线后续独立审核与发布材料。

限制：

- 未正式推进 Live 审核前，不要把 Live 能力写入 Clips 或 Support 站。
- 若未来用于 Riot 审核，需要独立 `riot.txt`。

### 3.3 `support.clutchreframe.com`

用途：

- Overwolf Store Listing 的 `Support link`。
- ClutchReframe Clips Dota 2-only app 的技术支持入口。
- 提供最小排障说明：Dota 2 setup、no clips / empty Library、recorder conflict、联系支持。

限制：

- 不做完整官网。
- 不放 `riot.txt`。
- 不写 LoL / Riot support。
- 不承诺 Premium、subscription、remove ads、cloud sync、自动上传、自动发布或 AI 剪辑。
- 不加载 analytics / tracking / third-party embeds。

## 4. GitHub Pages 与 DNS 配置

### 4.1 仓库

建议仓库：

```text
ClutchReframe/Clips-website
ClutchReframe/Live-website
ClutchReframe/Support-website
```

每个仓库独立开启 GitHub Pages，并在仓库根目录放置对应 `CNAME`。

### 4.2 CNAME 文件

`Clips-website/CNAME`：

```text
clips.clutchreframe.com
```

`Live-website/CNAME`：

```text
live.clutchreframe.com
```

`Support-website/CNAME`：

```text
support.clutchreframe.com
```

### 4.3 Cloudflare DNS

在 `clutchreframe.com` 的 DNS records 中添加：
https://www.cloudflare.com/

```text
clips    CNAME    clutchreframe.github.io
live     CNAME    clutchreframe.github.io
support  CNAME    clutchreframe.github.io
```

Cloudflare `Add record` 页面填写方式：

```text
Type: CNAME
Name: support
Target: clutchreframe.github.io
Proxy status: DNS only
TTL: Auto
```

要求：

- 不要使用 `A` record。`support.clutchreframe.com` 是 GitHub Pages 子域名，应使用 `CNAME`。
- Proxy status 使用 `DNS only`。
- TTL 使用 `Auto`。
- CNAME target 以 GitHub Pages 设置页实际提示为准。

说明：

- GitHub Pages 自定义域名初次签发 HTTPS 时，Cloudflare 橙云代理可能导致证书校验异常。
- 审核期间优先保持 DNS 路径简单，不使用额外缓存、转发、重写规则。

注意：如果 Cloudflare 登录页在普通 Chrome 窗口卡住，优先使用 Chrome 无痕窗口或 Edge 打开 `https://dash.cloudflare.com/login`。


## 5. GitHub Pages 设置

每个仓库：

1. Settings -> Pages。
2. Source 选择 `main` branch / root。
3. Custom domain 填对应子域名。
4. DNS 检查通过后启用 `Enforce HTTPS`。

Support 站设置：

```text
Custom domain: support.clutchreframe.com
Enforce HTTPS: enabled
```

## 6. 页面与文件要求

### 6.1 Clips / Live 审核站点

如果用于 Riot 审核，至少包含：

- `index.html`
- `privacy.html`
- `terms.html`
- `riot.txt`
- OG metadata / social preview image

`riot.txt` 要求：

- 放在站点根目录。
- `https://子域名/riot.txt` 可直接访问。
- 内容为单行验证字符串。
- 无 HTML 包裹、无 BOM、无多余空格。

### 6.2 Support 站点

Support 站点最小文件：

```text
index.html
privacy.html
terms.html
CNAME
.nojekyll
README.md
docs/2026-06-17_support_website_plan.md
```

Support 首页只保留：

- `ClutchReframe Clips Support`
- `Support for the Dota 2 Overwolf app`
- `clutchreframe@gmail.com`
- Dota 2 `-gamestateintegration` setup
- no clips / empty Library 基础排障
- recorder conflict 恢复步骤
- 联系支持时需要提供 app version、Overwolf version、截图、问题描述和复现步骤
- Privacy / Terms 链接

Support 首页不放：

- `riot.txt`
- LoL / Riot support
- 复杂 FAQ
- Diagnostics Export 详细教程
- Premium / subscription / remove ads
- cloud sync / 自动上传 / AI 剪辑
- analytics / tracking / third-party embeds

## 7. 审核期冻结规则

### Clips / Live

提交 Riot 或 Overwolf 审核后：

- 不改 Pages custom domain。
- 不改 DNS record。
- 不改核心 URL 结构：`/riot.txt`、`/privacy.html`、`/terms.html`。
- 内容可谨慎更新，但不要改变审核入口路径。

### Support

填入 Overwolf Developer Console 的 Support link 后：

- 不改 `https://support.clutchreframe.com/` 入口。
- 保持 `privacy.html` 和 `terms.html` 可访问。
- 不把 support 入口重定向到 `clips.clutchreframe.com`。
- 不加入与当前 Dota 2-only 提交冲突的 LoL / Riot 支持承诺。

curl -I https://clips.clutchreframe.com/terms.html

Support：

```bash
curl -I https://support.clutchreframe.com/
curl -I https://support.clutchreframe.com/privacy.html
curl -I https://support.clutchreframe.com/terms.html
```

Support 预期：

- 三个 URL 返回 `200 OK`。
- HTTPS 正常。
- 不跳转到 `clips.clutchreframe.com`。
- 页面不出现 LoL / Riot support 承诺。
- 页面不出现 Premium / subscription / remove ads / cloud sync / AI 剪辑承诺。
